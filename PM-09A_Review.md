# Module
PM-09A – AI Decision Timeline & Learning Engine

## Objective
Create an AI Decision Timeline that records how recommendations change over time. Every AI recommendation generated by the platform is stored as a historical snapshot, preserving the reasoning, confidence, and context behind each change.

## Files Changed
### Backend
- `backend/prisma/schema.prisma`: Added `AIDecisionTimeline` model.
- `backend/src/controllers/timeline.controller.ts` (NEW): Endpoints to fetch timeline data for a stock or across the user's portfolio.
- `backend/src/routes/timeline.routes.ts` (NEW): Router for the new endpoints.
- `backend/src/controllers/stockIntelligence.controller.ts`: Added logic to detect changes, request AI explanations for changes, and append snapshots to the timeline.
- `backend/src/index.ts`: Registered the `/api/timeline` route.

### Frontend
- `frontend/src/services/timeline.service.ts` (NEW): API client functions.
- `frontend/src/components/timeline/DecisionTimeline.tsx` (NEW): Timeline UI for individual stock history with expandable cards.
- `frontend/src/components/timeline/PortfolioTimelineDashboard.tsx` (NEW): Aggregated dashboard showing biggest upgrades, downgrades, stability, and volatility.
- `frontend/src/pages/dashboard/AIIntelligencePage.tsx`: Integrated the `DecisionTimeline` component at the bottom of the intelligence report.
- `frontend/src/pages/dashboard/PortfolioPage.tsx`: Added tabs to switch between the "Overview" and the new "AI Timeline & Insights" dashboard.

## Architecture Changes
- Implemented a cache-miss strategy: Whenever `stockIntelligence.controller.ts` regenerates a stock's data, it compares the new output against the existing snapshot.
- If the core recommendation changes, a secondary AI prompt is triggered to parse the "Why" (Technical, Fundamental, News, Macro, Valuation) and store it in strict JSON format.
- Frontend was updated with a tab-based navigation on the Portfolio page to accommodate the new dashboard without cluttering the main table view.

## Database Changes
Added the `AIDecisionTimeline` model to `schema.prisma`. This table is append-only and acts as the system of record for all AI decision events, tracking:
- Recommendation and overall score
- Change type (Initiated, Upgrade, Downgrade, Maintained)
- Detailed textual explanations for the changes
- Associated metadata (confidence and risk changes)

## API Changes
- `GET /api/timeline/:symbol`: Fetches the descending chronological history of snapshots for a specific stock.
- `GET /api/timeline/portfolio`: Fetches and computes the aggregated timeline data for the user's active portfolio (Biggest Upgrades, Biggest Downgrades, Most Stable, Highest Volatility).

## Security Changes
- Endpoints are protected via the existing `authenticate` middleware.
- Only the user's authenticated portfolio assets are used to aggregate the portfolio timeline data.

## Test Results
- Database model integrated successfully.
- Code implemented to gracefully handle AIEngine calls and fallback responses.

## Known Issues
- Because timeline snapshots rely on live market data changes, the timeline will only start populating once stocks are queried via the AI Intelligence tab over time.
- The `docker-compose` environment was not available during the build, so the Prisma schema push should be verified manually on deployment.
